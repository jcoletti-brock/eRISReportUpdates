# Technical Writing Standards and Best Practices

## Objective

Produce high-quality documentation for software products, technical concepts, or processes.
Compile a complete understanding of the subject matter, extract key information, and present it in a clear, structured format.
Perform research as needed to ensure accuracy and completeness.

## Directions and Best Practices

- **Clarity**: Use clear, concise, and unambiguous language.
- **Structure**: Organize content logically with headings, subheadings, and lists.
- **Audience**: Write for the intended audienceâ€™s knowledge level.
- **Voice**: Use active voice and direct instructions.
- **Jargon**: Avoid jargon unless necessary, and define technical terms.
- **Consistency**: Be consistent with terminology, formatting, and style.
- **Visuals**: Include examples, illustrations, or diagrams where helpful.
- **Review**: When making changes, review and update existing parts of the documentation for accuracy.
- **Proofreading**: Proofread for grammar, spelling, and clarity.

## Output Format

### Output Directory

- Clarify where the output documents should be placed within the workspace before beginning.
- Do not assume the output location unless specified in a previous task.

### Output Type

- Documents should be in Markdown format.

### Document Structure

- **Title**: Clear and descriptive title.
- **Introduction**: Brief overview of the topic or purpose of the document.
- **Body**: Detailed information organized into sections and subsections.
- **Conclusion**: Summary of key points and any final thoughts.

## Tools and Resources

- **Research Tools**: Web searches, documentation, and other resources to gather information.
- **Mermaid Diagrams**: Use Markdown Mermaid syntax for flowcharts, diagrams, and visual representations.
